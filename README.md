# 📖 mh-ts-manager

**Диспетчер пользовательских сессий** (v1.3.0) — десктопное приложение на C# 12 / .NET 8 / WinUI 3 для администраторов и специалистов технической поддержки. Предназначен для централизованного просмотра, управления и удалённого взаимодействия с пользовательскими сессиями на компьютерах под управлением **Windows 10/11 (x64)**.

> ⚠️ **Только Windows.** Кроссплатформенная поддержка (Linux, macOS) не планируется.

---

## ✨ Возможности

### 🔍 Просмотр и управление сессиями

- **Список сессий** в стиле Windows 11 Task Manager (вкладка «Пользователи»)
- **Композитная ячейка пользователя:** `Логин — Полное имя (приложений: N)` с иконками 🔒/🖥️
- **Статусы сессий:** Активна, Бездействие, Отключена, Заблокирована (не агрегированные состояния процессов!)
- **Столбцы:** Пользователь, Состояние, ЦП, Память, Диск, Сеть, GPU, Движок GPU (+ скрытые: ID, Сеанс, Имя клиента, NPU)
- **Управление столбцами:** drag-and-drop, «Выбрать столбцы», сохранение конфигурации

### 🔗 Удалённое подключение

- Двойной клик или кнопка → запуск `mstsc /shadow:<ID> /control`
- Контекстное меню: Развернуть, Переключиться, Отключить, Выйти, Отправить сообщение

### 🪟 Список приложений

- При раскрытии сессии — список **окон**, видимых в `Alt+Tab` (не процессов!)
- Фильтрация через `EnumWindows` + `ProcessIdToSessionId` + проверка стилей окон
- Иконка приложения + заголовок окна

### 🔐 Контроль привилегий

- Кнопка-замок: индикация режима (`🔓` = админ, `🔒` = ограниченный)
- Повышение прав через UAC (`runas`) с сохранением состояния
- Автоматическая блокиция действий, требующих прав администратора

### 🌐 Локализация

- Поддержка всех языковых пакетов Windows
- Автоматический fallback: текущий → `ru-RU` → `en-US`
- RTL-поддержка для `ar-SA`, `he-IL`, `fa-IR`
- 100% текстов вынесены в `.resw`, харкод отсутствует

### ⚡ Оптимизации

- **Асинхронный опрос** каждые 5 сек без блокировки UI
- **Виртуализация списков** для плавной работы при 50+ сессиях
- **Идемпотентность команд:** проверка состояния перед выполнением
- **SafeHandle** для всех P/Invoke вызовов, CancellationToken для фоновых задач

## 🚀 Установка

### 1. Self-contained EXE (без установки)

1. Скачать `mh-ts-manager-win-x64.zip` из [Releases](https://github.com/pavelvkovalenko/mh-ts-manager/releases)
2. Распаковать в любую директорию
3. Запустить `mh-ts-manager.exe`

**Требование:** Не требуется установленный .NET Runtime

### 2. MSIX пакет (рекомендуется)

1. Скачать `mh-ts-manager_1.3.0.0_x64.msix`
2. Двойной клик → `Установить` (или через PowerShell: `Add-AppxPackage`)
3. Запуск из Start Menu: «Диспетчер пользовательских сессий»

**Требование:** Developer Mode или доверенный сертификат

### 3. Сборка из исходников

```powershell
git clone https://github.com/pavelvkovalenko/mh-ts-manager.git
cd mh-ts-manager

# Сборка — скрипт проверит зависимости и запросит разрешение на установку
.\_build.ps1 -Configuration Release

# Автоматическая установка зависимостей без запроса
.\_build.ps1 -Configuration Release -Yes

# Только сборка, без проверки зависимостей
.\_build.ps1 -Configuration Release -SkipDependencyCheck
```

**Требования:**
- .NET 8 SDK (установится автоматически при согласии)
- Visual Studio 2022 17.8+ с workload «Windows App SDK» (опционально)

## ⚙️ Конфигурация

### Файл настроек

Путь: `%APPDATA%\mh-ts-manager\settings.json`

```json
{
  "general": {
    "language": "system",
    "theme": "system",
    "autoRefreshInterval": 5,
    "expandAppsByDefault": false
  },
  "ui": {
    "sessions": {
      "columns": {
        "visible": ["User", "Status", "CPU", "Memory", "Disk", "Network", "GPU", "GPUEngine"],
        "order": ["User", "Status", "CPU", "Memory", "Disk", "Network", "GPU", "GPUEngine", "SessionID", "SessionType", "ClientName", "NPU", "NPUEngine"]
      }
    }
  },
  "advanced": {
    "enableDebugLogging": false,
    "logRetentionDays": 7,
    "confirmDangerousActions": true
  }
}
```

### Аргументы командной строки

```
mh-ts-manager.exe [ОПЦИИ]
```

| Аргумент | Описание | Пример |
|----------|----------|--------|
| `--config <path>` | Путь к альтернативному файлу настроек | `--config "C:\custom\settings.json"` |
| `--language <code>` | Принудительный язык интерфейса | `--language ru-RU` |
| `--theme <mode>` | Принудительная тема | `--theme dark` |
| `--debug` | Включить отладочное логирование | `--debug` |
| `--help`, `-h` | Вывод справки | `--help` |
| `--version`, `-v` | Вывод версии | `--version` |

> ⚠️ Аргументы командной строки имеют приоритет над файлом конфигурации.

## 🔧 Управление и диагностика

### Запуск

```powershell
# По умолчанию
mh-ts-manager.exe

# Тёмная тема, русский интерфейс
mh-ts-manager.exe --theme dark --language ru-RU

# Отладочное логирование
mh-ts-manager.exe --debug
```

### Логи

- **Путь:** `%APPDATA%\mh-ts-manager\logs\`
- **Формат:** `mh-ts-manager-YYYYMMDD.log` (ротация по дате)
- **Макс. размер:** 5 МБ на файл
- **Уровни:** `INFO`, `WARN`, `ERROR`, `DEBUG` (только при `--debug`)

```powershell
# Просмотр последних логов
Get-Content "$env:APPDATA\mh-ts-manager\logs\*.log" -Tail 50

# Фильтрация ошибок
Get-Content "$env:APPDATA\mh-ts-manager\logs\*.log" | Select-String "ERROR|WARN"
```

### Типичные проблемы

| Проблема | Решение |
|----------|---------|
| Приложение не запускается | Проверить `.NET 8 Desktop Runtime`: `dotnet --list-runtimes` |
| Пустой список сессий | Запустить с правами администратора |
| Не работают кнопки «Отключить»/«Выйти» | Требуется режим администратора (клик на 🔒) |
| Неправильный язык интерфейса | Удалить `settings.json` или указать `--language ru-RU` |
| Зависание при обновлении | Запустить с `--debug`, проверить логи |

## 🛡️ Безопасность

- **Безопасные вызовы:** Все внешние процессы (`mstsc`, `msg`) через `ProcessStartInfo` с валидацией
- **Защита от symlink-атак:** Проверка путей через `Path.GetFullPath()` + `FileAttributes.ReparsePoint`
- **Минимальные привилегии:** По умолчанию без прав администратора
- **Без PII в логах:** Не логируются пароли, имена пользователей, IP-адреса
- **SafeHandle:** Автоматическая очистка всех P/Invoke хендлов
