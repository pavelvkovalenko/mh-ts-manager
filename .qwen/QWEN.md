## 📋 О проекте

**mh-ts-manager** — Диспетчер пользовательских сессий для Windows 10/11 (x64).

- **Технологии:** C# 12 / .NET 8 / WinUI 3 (Windows App SDK)
- **Назначение:** Просмотр, управление и удалённое подключение к пользовательским сессиям
- **Исполняемый файл:** `mh-ts-manager.exe`
- **Лицензия:** Проприетарная / По соглашению
- **Автор:** Коваленко Павел
- **Репозиторий:** `https://github.com/pavelvkovalenko/mh-ts-manager`

## 📝 Правила кода (кратко)

### Язык
- **Комментарии в коде:** русский
- **Логи:** английский
- **Пользовательские сообщения:** `.resw` ресурсы (без хардкода)
- **Имена:** английский (`PascalCase` методы, `_camelCase` поля)
- **XML-документация:** русский

### Безопасность
- **Всегда:** `ProcessStartInfo` с валидацией, `SafeHandle`, `CancellationToken`
- **Никогда:** `Process.Start(string)`, `.Result`/`.Wait()`, `async void` (кроме event handlers)
- **Защита:** `Path.GetFullPath()`, проверка `ReparsePoint`, санитизация ввода

### Управление ресурсами
- **`using var`** для всех `IDisposable`
- **`SafeHandle`** для P/Invoke хендлов
- **`CancellationToken`** передаётся по всей цепочке async-методов
- **`ArrayPool<T>`** для временных буферов

### Git-воркфлоу
- **Ветки:** `feature/...`, `fix/...`, `refactor/...`
- **Коммиты:** русский, `[тип]: описание` (`feat:`, `fix:`, `docs:`, `test:`)
- **PR:** подробное описание на русском + раздел `## ✅ Тестирование`

## 🤖 Подсказки для AI-ассистента

### При генерации кода
1. **Всегда** использовать `async/await`, `CancellationToken`, `using var`
2. **Всегда** использовать `SafeHandle` для P/Invoke
3. **Никогда** не хардкодить строки — использовать `.resw` ресурсы
4. **Комментарии** на русском, **логи** на английском
5. **Имена** переменных/методов на английском (`PascalCase`/`_camelCase`)
6. **XML-документация** (`/// <summary>`) на русском

### При рефакторинге
1. Сохранять существующую архитектуру и паттерны
2. Следовать правилам из `docs/development/RULES.md`
3. Предлагать улучшения, но не ломать существующий API

### При создании тестов
1. Формат именования: `Method_Scenario_ExpectedResult`
2. Структура: Arrange-Act-Assert
3. Использовать Moq для моков интерфейсов

### При работе с документацией
1. Все тексты на русском (кроме технических терминов)
2. Ссылаться на соответствующие разделы ТЗ
3. Обновлять версию документа при значительных изменениях

## ⚠️ Ограничения

- **Только Windows 10/11 x64** — не поддерживать Linux, macOS, ARM64
- **WinUI 3 предпочтительно** — WPF только как альтернатива
- **.NET 8** — не использовать более ранние версии
- **Минимум NuGet-зависимостей** — максимум встроенных средств
